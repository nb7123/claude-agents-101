# 代码审查清单

## 安全性
- [ ] 没有硬编码密钥/密码/token
- [ ] 用户输入有验证和消毒
- [ ] 没有 SQL 注入/命令注入风险
- [ ] 敏感数据不写入日志

## 可靠性
- [ ] 所有 async 操作有错误处理
- [ ] 边界条件有覆盖（空数组、undefined、极值）
- [ ] 没有未处理的 Promise rejection
- [ ] 资源（文件句柄、连接）有正确释放

## 可维护性
- [ ] 函数职责单一，长度 < 50 行
- [ ] 变量名清晰表达意图
- [ ] 复杂逻辑有注释说明"为什么"，而非"做什么"
- [ ] 没有重复代码（DRY 原则）

## 性能
- [ ] 循环内没有不必要的 await
- [ ] 独立异步操作用 Promise.all 并发
- [ ] 大数据集用流式处理，不一次性加载到内存

## 测试
- [ ] 核心逻辑有单元测试
- [ ] 边界条件有测试覆盖
- [ ] 测试用例命名清晰（given/when/then 格式）
